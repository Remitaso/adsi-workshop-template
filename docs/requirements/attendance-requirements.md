# 勤怠管理アプリ — 要求仕様書

## 1. プロダクト概要

社員の出退勤打刻・休暇管理・勤務時間集計を行う Web アプリケーション。
約 500 名規模の組織で、一般社員・管理者・人事部門が利用する。

---

## 2. ユーザーロール

| ロール | 説明 |
|--------|------|
| 一般社員 | 自分の打刻・休暇申請・勤怠確認を行う |
| 管理者（上長） | 部下の打刻修正承認・休暇承認・チーム勤怠確認 |
| 人事部門 | 全社員の勤怠集計・レポート出力・マスタ管理 |

---

## 3. 組織構造

- 3 階層: **部 > 課 > チーム**
- 社員は 1 つのチームに所属する
- 管理者は課またはチーム単位で承認権限を持つ

---

## 4. ユーザーストーリー

### 4.1 打刻

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-01 | 社員として、Web 画面のボタンで出勤を打刻したい | 出勤ボタン押下で現在時刻が記録される |
| US-02 | 社員として、Web 画面のボタンで退勤を打刻したい | 退勤ボタン押下で現在時刻が記録される |
| US-03 | 社員として、1日に複数回の出退勤（中抜け後の再出勤）をしたい | 再出勤が可能で、各出退勤ペアが記録される |
| US-04 | 社員として、打刻を修正したい（上長承認前は自由に修正可能） | 未承認の打刻は自由に修正でき、修正履歴が残る |
| US-05 | 管理者として、部下の打刻修正を承認したい | 承認すると打刻が確定し、以降修正不可になる |

### 4.2 勤務時間計算

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-06 | 社員として、日次の勤務時間が自動計算されるのを確認したい | 出退勤ペアから実労働時間が自動算出される |
| US-07 | システムとして、8 時間超を残業として計算する | 日 8 時間超分が残業時間として集計される |
| US-08 | システムとして、深夜帯（22:00〜5:00）を深夜割増として計算する | 深夜帯の勤務時間が区別して記録される |
| US-09 | システムとして、週 40 時間を所定労働時間として管理する | 週次集計で 40 時間超が残業に算入される |

### 4.3 休暇管理

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-10 | 社員として、休暇を申請したい（種類: 有給・半休・代休・特別休暇・欠勤） | 休暇種類を選択して申請でき、管理者に通知される |
| US-11 | 管理者として、部下の休暇申請を承認/却下したい | 承認・却下操作ができ、社員に結果が通知される |
| US-12 | システムとして、有給休暇を勤続年数に応じて自動付与する | 法定基準に従い年次で有給が付与される |
| US-13 | 社員として、自分の有給残日数を確認したい | 付与日数・取得日数・残日数が表示される |

### 4.4 集計・レポート

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-14 | 人事として、月次の勤怠集計を確認したい | 月間労働時間・残業時間・休暇取得日数が一覧表示 |
| US-15 | 人事として、勤怠データを CSV エクスポートしたい | 月次集計を CSV ファイルとしてダウンロードできる |
| US-16 | 管理者として、チームの勤怠状況をダッシュボードで確認したい | チーム全員の出勤状況・残業状況が一覧表示される |

### 4.5 アラート・通知

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-17 | システムとして、月 45 時間超の残業を警告したい | 該当社員と管理者にアラートが表示される |
| US-18 | システムとして、打刻忘れを通知したい | 出勤日に打刻がない場合、翌営業日に通知される |

### 4.6 認証・管理

| # | ストーリー | 受け入れ基準 |
|---|-----------|-------------|
| US-19 | 社員として、ID/パスワードでログインしたい | ログイン成功で自分のダッシュボードが表示される |
| US-20 | 人事として、社員マスタ（部署・ロール）を管理したい | 社員の追加・編集・無効化ができる |

## 5. 勤務形態

| 項目 | 仕様 |
|------|------|
| 勤務形態 | フレックスタイム制 |
| 所定労働時間 | 週 40 時間 |
| コアタイム | （要確認: 別途設計フェーズで決定） |
| 残業基準 | 日 8 時間超 |
| 深夜割増 | 22:00〜5:00 |
| 残業上限アラート | 月 45 時間 |

---

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| プラットフォーム | Web アプリケーション（レスポンシブ不要） |
| 認証 | ID / パスワード認証 |
| 想定同時利用者 | 〜500 名 |
| データ保持期間 | 法定 3 年間 |
| 給与連携 | 不要（CSV エクスポートで対応） |

## 8. MVP スコープ

全機能が MVP に含まれる:

- [x] 出退勤打刻（再出勤対応）
- [x] 打刻修正（承認フロー付き）
- [x] 勤務時間の自動計算（残業・深夜割増）
- [x] 休暇申請・承認
- [x] 月次集計レポート
- [x] CSV エクスポート
- [x] 残業アラート（月 45 時間）
- [x] 管理者ダッシュボード

---

## 9. 用語集

| 用語 | 定義 |
|------|------|
| 打刻 | 出勤・退勤の時刻を記録する操作 |
| 打刻ペア | 1 回の出勤〜退勤の組。1 日に複数ペア可 |
| 確定 | 管理者が承認し、修正不可になった状態 |
| フレックスタイム | コアタイム以外は出退勤時刻を社員が自由に決められる制度 |
| 所定労働時間 | 週 40 時間。超過分が残業 |
